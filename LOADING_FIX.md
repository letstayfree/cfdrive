# CFDrive 加载问题修复指南

## 问题描述
访问应用时看到登录页面一直显示加载动画，无法正常使用。

## 原因分析
数据库迁移尚未应用，导致后端查询数据库时失败。

## 解决方案

### 方法一：使用批处理脚本（推荐）
1. 在项目根目录找到 `migrate.bat` 文件
2. 双击运行该文件
3. 等待迁移完成
4. 刷新浏览器

### 方法二：手动执行命令
打开新的 PowerShell 窗口，执行以下命令：

```powershell
cd d:\Projects\CF\cfdrive\packages\worker
wrangler d1 migrations apply cfdrive-db --local --persist-to ../../.wrangler/state
```

### 方法三：重启开发服务器
1. 停止当前所有 `npm run dev` 进程
2. 在项目根目录执行 `npm run db:migrate`
3. 然后重新启动 `npm run dev`

## 验证修复

修复后，访问 `http://localhost:5173` 应该能看到：
- 如果是首次安装：显示初始设置页面
- 如果已有用户：显示登录页面（不再一直加载）

## 已修复的代码

已改进前端的错误处理逻辑，即使数据库查询失败，也会显示登录页面而不是无限加载。

## 常见问题

### Q: 运行迁移时提示错误
A: 确保没有其他 wrangler 进程在运行。可以先停止所有开发服务器，然后单独运行迁移。

### Q: 迁移完成后仍然加载
A: 刷新浏览器并清除缓存（Ctrl+Shift+R）

### Q: 提示 "系统需要初始设置"
A: 这是正常的，说明迁移成功。按照页面提示创建管理员账户即可。
